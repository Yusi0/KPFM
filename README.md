# KPFM Leaderboard (KPFM 리더보드)

KPFM 클랜의 경기 기록을 관리하고 시각화해 주는 웹 기반 개인 전적 및 랭킹 대시보드입니다. PFAttach 스타일의 모던한 글래스모피즘(Glassmorphism) 디자인 시스템이 적용되어 있으며, 모바일 최적화 레이아웃으로 설계되어 기기에 상관없이 아름다운 화면을 보여줍니다.

## 🔗 라이브 데모

## ✨ 핵심 기능

### 1. 랭킹 대시보드 (`index.html`)
* **모던 카드형 랭킹**: 테이블 방식에서 벗어나 랭킹 순위, 프로필 아바타, 티어 배지(Platinum -> Plat 단축 적용)와 함께 핵심 스탯(ELO, KDR, 승률, 경기 수)을 2x2 그리드로 한눈에 보여줍니다.
* **실시간 유저 검색**: 스마트 검색창을 통해 닉네임을 타이핑하면 자동 완성 완성 목록이 표시되어 빠르게 특정 유저를 필터링할 수 있습니다.
* **팀 메이킹 로비 (Team Balance)**:
  * 리더보드 카드에서 `+` 버튼을 누르거나 검색창에서 플레이어를 선택하면 상단 로비 리스트에 실시간 추가됩니다.
  * 로비에 추가된 플레이어는 **리더보드 최상단으로 정렬**되어 현재 모집된 스쿼드를 한눈에 파악하기 편리합니다.
  * **균등 팀 배정 알고리즘**: ELO 점수를 기반으로 두 팀(Phantom, Ghost)의 평균 ELO 편차가 가장 적도록 균형감 있게 스쿼드를 자동 분할해 줍니다.
* **글래스모피즘 테마 및 배경 블롭 애니메이션**: 화면 테마(다크 모드 / 라이트 모드) 변경이 즉시 동적으로 반영되며 뒷배경의 흐르는 빛무리(Blob)가 시각적 깊이감을 더해줍니다.

### 2. 플레이어 상세 분석 페이지 (`user.html`)
* **프로필 요약**: 현재 순위, ELO 점수, 전체 KDR 및 게임당 킬(KPR), 전적(승/패/승률), 총 누적 매치 정보를 요약 카드에 표시합니다.
* **트렌드 시각화 차트 (Chart.js)**:
  * **ELO Trend**: 매 라운드를 진행하며 변동된 ELO의 추이를 그래프로 렌더링합니다. 마우스 호버 시 상세 티어도 툴팁으로 함께 출력됩니다.
  * **KDR Trend**: 플레이어의 실시간 누적 KDR 변동 추이를 시각화하여 실력의 성장 곡선을 확인하게 해줍니다.
  * 다크/라이트 테마 전환 시 그리드 라인과 폰트 색상이 동기화됩니다.
* **맵별 전적 통계**: 각 맵(Metro, Warehouse, Desert Storm, Elevation)에서의 플레이 횟수, 승리 횟수, 그리고 승률(50% 기준 컬러 하이라이트)을 표로 보여줍니다.
* **상세 매치 기록 및 팝업 모달**:
  * 최근 치른 경기 라운드, 킬, 데스, 당시 KDR, ELO 변동 수치를 연대순으로 정렬하여 출력합니다.
  * 개별 매치 행을 클릭하면 **라운드 디테일(Round Details) 모달**이 열리며, 양 팀(Phantom vs Ghost)의 킬, 데스 기록 및 최종 ELO가 시각적으로 출력됩니다.
  * **팀별 MVP 강조**: 각 팀에서 해당 라운드 동안 가장 좋은 ELO 성과를 낸(가장 많이 상승했거나 가장 적게 하락한) 플레이어를 **팀별 MVP**로 선정하여 골드 하이라이트로 세련되게 보여줍니다.

## 📂 파일 구조 및 데이터 구성

본 프로젝트는 데이터베이스가 필요 없는 **서버리스(Serverless)** 정적 파일 구조로 설계되었으며, `lboard.csv`, `match.csv`, `aftercalc.csv` 3개의 CSV 파일을 실시간 페치하여 구동됩니다.

```bash
Leaderboard/
├── index.html       # 메인 랭킹 및 팀 배정 대시보드
├── user.html        # 유저 전적 상세 분석 및 차트
├── lboard.csv       # 누적 플레이어 랭킹 요약 데이터
├── match.csv        # 라운드별 모든 플레이어 개인 인게임 스탯 데이터
└── aftercalc.csv    # 라운드 종료 직후 실시간 계산된 ELO 기록 데이터
```

### 📊 데이터 갱신 방법 (엑셀/구글 시트 연동)
전적 기록을 새로 업데이트하고 싶다면 엑셀이나 구글 스프레드시트에서 데이터를 편집한 뒤, 해당 파일명 그대로 `.csv` 형식으로 프로젝트 루트 경로에 덮어씌우면 됩니다.

1. **`lboard.csv`**:
   * 헤더: `Nickname,Kills,Deaths,Wins,Losses,Total,ELO,Tier`
   * 모든 경기 통합 누적 통계값을 나타냅니다.
2. **`match.csv`**:
   * 헤더: `Round,Map,Nickname,Kills,Deaths,Team,Winning Team`
   * 매 라운드당 각 플레이어의 활약상을 기록합니다. `Team`은 플레이어가 속했던 팀(1: Phantom, 2: Ghost)이며, `Winning Team`과 일치하면 승리로 집계됩니다.
3. **`aftercalc.csv`**:
   * 헤더: `Round,Nickname,ELO`
   * 각 라운드 종료 시점에서의 ELO 레이팅 수치 변화를 역대 라운드 순서대로 누적 기록합니다. 프론트엔드는 이 데이터를 통해 `previous ELO`와 비교하여 한 경기에서의 ELO 상승/하락 폭을 산출합니다.

## 💻 로컬 실행 방법

브라우저 보안 정책(CORS)으로 인해 로컬에서 CSV 파일을 읽어오려면 가벼운 웹 서버(예: `http-server`)를 통해 실행해야 합니다.

1. 프로젝트 디렉토리로 이동합니다.
2. 아래 명령어로 로컬 서버를 구동합니다:
   ```bash
   npx http-server -p 8080 -c-1
   ```
   *(참고: `-c-1` 옵션은 브라우저 캐싱을 비활성화하여 코드가 즉각 업데이트되도록 돕습니다.)*
3. 크롬 등 브라우저에서 `http://localhost:8080/`으로 접속합니다.
